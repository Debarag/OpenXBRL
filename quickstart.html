Quick Start

This is a quick guide on how to get started with the OpenXBRL package.

Installation

The package is not yet available as a pip install, but a manual install is possible.

To install, copy the /openxbrl folder into your python working directory. This allows you to import the package.

However, your python environment may not have the needed dependent packages to run OpenXBRL. To install those

  1. get the requirements.txt file from the OpenXBRL project and
  2. run pip install -r /[pathto]/requirements.txt where [pathto] is the location of the requirements.txt file.

Usage

To use OpenXBRL, you can import it into your Python code like this:

from openxbrl import OpenXBRL

OpenXBRL has an API to get a structured set of accounting elements for a particular corporate filing. The API call requires the CIK number for the company, the fiscal year, and the fiscal quarter. The fiscal quarter can be 1, 2, 3, or 0 where 0 indicates the annual (10-K) filing. Form 10-Q data are reported for the first, second, and third quarters. The API looks for both regular and amended filings (10-K/A and 10-Q/A). If no filing is found, the API call returns a blank response.

Sample code: 


    from openxbrl import OpenXBRL

    api = OpenXBRL('./workingdir')
    filing = api.get_filing_accounting( cik=1705, fiscal_year=2022, fiscal_quarter=3)
    print( filing )
The OpenXBRL() initialization requires a working directory to which downloaded files can be written. The function returns a python dict which holds the filing's accounting parameters.
